
### Початкова таблиця order_report

| Атрибут | Опис |
|---|---|
| order_id | Ідентифікатор замовлення |
| order_date | Дата замовлення |
| user_name | Ім’я користувача |
| user_email | Електронна пошта користувача |
| user_phone | Номер телефону |
| country | Країна доставки |
| city | Місто доставки |
| street | Вулиця |
| house_number | Номер будинку |
| postal_code | Поштовий індекс |
| product_name | Назва товару |
| category_name | Назва категорії |
| quantity | Кількість товару |
| price | Ціна товару |
| payment_method | Спосіб оплати |
| payment_status | Статус оплати |

Початкова структура містить дублювання даних та може призводити до аномалій вставки, оновлення та видалення.

---

## Функціональні залежності

Для початкової таблиці можна виділити наступні функціональні залежності:

- user_email → user_name, user_phone
- product_name → price, category_name
- order_id → order_date, payment_method, payment_status
- category_name → category_description

---

## Перша нормальна форма (1NF)

Для приведення бази даних до першої нормальної форми були усунені повторювані групи даних та забезпечено атомарність значень.

Кожне поле таблиці містить лише одне значення, а всі записи мають унікальний первинний ключ.

---

## Друга нормальна форма (2NF)

Для приведення бази даних до другої нормальної форми були усунені часткові залежності атрибутів від первинного ключа.

Дані користувачів, товарів та категорій були винесені в окремі таблиці:

- users
- products
- categories

---

## Третя нормальна форма (3NF)

Для приведення бази даних до третьої нормальної форми були усунені транзитивні залежності шляхом розділення даних на окремі таблиці.

Адреси доставки, оплата та позиції замовлень були винесені в окремі таблиці:

- addresses
- payments
- order_items

---

## Кінцева схема бази даних

Фінальна структура бази даних містить наступні таблиці:

- users
- addresses
- categories
- products
- orders
- order_items
- payments

Між таблицями встановлені зв’язки за допомогою первинних та зовнішніх ключів.
